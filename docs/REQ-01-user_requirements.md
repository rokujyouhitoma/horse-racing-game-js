# [REQ-01] 要求定義書 (User Requirements Document) - horse-racing-game-js

本ドキュメントは、ブラウザベースの競馬ボードゲーム「`horse-racing-game-js`」におけるユーザー要求（User Requirements）を定義したものです。プレイヤーがどのような動機でゲームをプレイし、どのような体験や操作を期待するかを整理します。

---

## 1. プロジェクト背景と目的 (Background & Objectives)

### 1.1 背景 (Background)
アナログのボードゲームやカードゲームは戦略的で面白い一方、物理的なセットアップ（カードシャッフル、フィギュアやトラックの配置、得点計算）に手間がかかる課題があります。特に、プレイヤー間の戦略や心理戦が中心となる競馬ボードゲームでは、複雑な移動効果やオッズ計算を手動で行うのが難しく、デジタル化による効率向上が望まれていました。

### 1.2 目的 (Objectives)
本プロジェクトは、手軽にブラウザで遊べる戦略的競馬ボードゲームのデジタル版を提供します。プレイヤーは「オッズを見てコインをベットする楽しさ」と「手札から戦略カードをプレイして意中の馬をゴールさせる駆け引き」を、物理コンポーネントを準備することなく直感的に楽しむことができます。

---

## 2. ターゲットユーザーおよび想定ユースケース (Target Users & Use Cases)

* **ターゲットユーザー**:
  * アナログボードゲームや戦略カードゲームが好きな愛好家。
  * 面倒なアプリのインストールやログインなしに、ブラウザでサクッと頭脳対戦ゲームを楽しみたいユーザー。
  * 乱数シミュレーションや決定論的なゲームシステム（運要素の制御、リプレイ等）に関心のあるデバッガーやゲームデザイナー。

* **主要ユースケース**:
  1. **ゲーム開始（ベットフェーズ）**:
     * 画面上の出走馬（モンスター）のオッズテーブルを確認し、高配当を狙うか堅実に本命に賭けるか検討してコインをベットする。
  2. **カードプレイとレース進行**:
     * 手札のカードを引き、自ターンに1枚カードを選択してプレイする。
     * プレイされたカードの効果（Step、Rank、Dash）により、馬がボード上（70マス）を前進していく様子をグラフィカルに視認する。
  3. **レース終了と配当受け取り**:
     * 1着および2着の馬がゴールライン（70マス超）に到達するとレースが終了し、オッズに従ってコイン配当が自動計算される。
  4. **一手戻す（Undo機能 - デバッグ/練習）**:
     * レース中にプレイされたカード効果が気に入らなかったり、別の戦術を検証したい場合に、「一手戻す（Undo）」ボタンを押下して直前の状態に巻き戻してゲームをやり直す。
  5. **自動実行・シミュレーション（デバッグモード）**:
     * 自分でカードを選んで進めるのではなく、山札から自動的にカードがプレイされていく様子をオートで高速再生し、ゲームバランスを検証する。
