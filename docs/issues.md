# 🏇 horse-racing-game-js 課題・改善ポイント定義書

`horse-racing-game-js` のソースコードおよび開発ノートを分析して得られた、バグ、未実装機能、今後の提案などをまとめたドキュメントです。

---

## ⚠️ 課題・未実装機能 (Issues & TODOs)

### 1. 🟥 【重要・バグ】プレイカードを使い切っても2位が決まらずにスタックする問題
* **現象**: プレイカードをすべて使い切った場合、本来は1着・2着の馬が必ず決定するはずのゲームバランスになっているが、特定のカード順で実行された場合、1位は決まるものの **2位が未決定のままゲームがスタック（あるいは2位決定処理が正常にトリガーされない）** してしまう。
* **原因の仮説**: 
  * `RaceDirector.prototype.OnUpdate` のゴール判定ロジックが、同時にゴールした場合や一部の特殊効果適用時、また `Undo` を挟んだ際に正しく `goals_` 配列を処理できていない。
  * 特殊カード（Dash/Rank）による同着時の挙動や、カードの山札（全60枚）のレベルデザイン上のバランスと判定条件のミスマッチ。

### 2. 🟨 【未実装】ゲームシステムの本実装
現在はレースをデバッグ自動プレイする機能や、スタブデータでの単純なシミュレーションがメインとなっており、ボードゲームとしての「対戦プレイ」の枠組みが未実装です。
* **プレイヤー管理**: プレイヤー人数の決定、各プレイヤーの担当モンスターの決定、プレイヤーごとのターン処理。
* **ベットシステム**: レース開始前にモンスターコインをベットする仕組み（`Bet` クラスは仮実装されているが、UIや選択ロジックが未実装）。
* **手札配布と山札制御**: プレイヤーにカードを手札として配布し、山札から引いて、手札から選んでプレイする一連のゲームサイクル。
* **ポイント計算**: レース終了時の着順オッズに基づく最終ポイント/コイン計算とゲーム全体の勝敗判定。

### 3. 🟨 【未実装】マスターデータのバリューチェッカー
* `RelationshipChecker`（関係性チェック）は実装されていますが、`ValueChecker`（値自体の範囲や正当性をバリデーションする機構）が TODO のまま未実装となっています。

### 4. 🟦 【設計】密結合の解消とリファクタリング
* **レンダラーの密結合**: レンダラー（`RacetrackLayer` や `OddsTableLayer` など）がゲームの内部オブジェクトやディレクターと非常に強く結合しています。ゲームモデルの変更が描画レイヤーに直接影響するため、クライアント/サーバーモデルのような、より疎結合なデータ受け渡し設計への移行が望まれます。
* **コーディング規約の欠如**: 開発時期の違いやルールの未整備により、キャメルケース/スネークケースの混在や、オブジェクト指向の継承方法に一貫性がない（例: `GameObject` の継承スタイルなど）ため、整理が必要です。
* **エンジンラグ処理時の不整合**: バックグラウンド復帰時のラグ解消処理（whileループのスキップ）により、シミュレーションゲームとしての状態更新に不整合が生じる恐れがあります。

## 🚀 今後の進め方（提案）

1. **バグの修正**: プレイカード使い切り時に2位が決定しない不具合を再現・特定し、ゴール判定アルゴリズムの堅牢化を図る。
2. **ベット・ゲームサイクルの実装**: プレイヤーのターン制御、手札からカードを選択して出すUIの構築、オッズに基づくコインのベットとリザルトへの反映。
3. **リファクタリング**: `Render` 部分と `Model`/`Director` の連携を Pub/Sub イベント経由に統一し、密結合を緩和する。
