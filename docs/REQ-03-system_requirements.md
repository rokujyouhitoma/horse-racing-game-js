# [REQ-03] 要件定義書 (System Requirements Document) - horse-racing-game-js

本ドキュメントは、「`horse-racing-game-js`」におけるシステム要件および技術仕様を定義します。開発者が実装・デバッグを行う上での詳細な数値要件、機能要件、および制約条件を明記します。

---

## 1. システム構成・動作環境要件

* **アーキテクチャスタイル**: 完全クライアントサイドの静的シングルページアプリケーション (SPA)。サーバーサイドのデータベースやAPI通信を行わず、すべての状態更新および画面描画をローカルブラウザ内で完結する。
* **主要な技術スタック**:
  * コア: HTML5 / CSS3 / Vanilla JavaScript (ES6)
  * ビルドツール: Google Closure Compiler (コード縮小・アノテーションバリデーション用)
  * フォント: Google Fonts (`Outfit`, `Inter`, `Noto Serif JP`)
* **稼働ターゲット**: WebGL非依存の2D DOM/Canvas描画。最新のモダンブラウザ (Chrome, Edge, Firefox, Safari) が動作するスマートフォン、タブレット、PC。

---

## 2. マスターデータおよびデータ検証要件

ゲームは外部データベースの代わりに、JavaScriptコード内にハードコードされたCSVスキーマ（二次元配列）をマスターデータとして読み込みます。

### 2.1 データモデル構成
* **StepCardDetail**: `id (int)`, `color (string)`, `step (int)`
* **RankCardDetail**: `id (int)`, `rank (int)`, `step (int)`
* **DashCardDetail**: `id (int)`, `name (string)`, `type (int)`
* **PlayCard**: `id (int)`, `type (int: Step=1, Rank=2, Dash=3)`, `detail_id (int)`

### 2.2 リレーションチェック要件 (`RelationshipChecker`)
データ読み込み時に、以下の整合性を自動で検証し、エラーがあれば開発者コンソールへ出力すること。
- `PlayCard.detail_id` が `PlayCard.type` に対応する各カード詳細 (`StepCardDetail.id`, `RankCardDetail.id`, `DashCardDetail.id`) に存在していること。

### 2.3 値検証チェック要件 (`ValueChecker`)
データ読み込み時に、以下の値の範囲や正当性を自動検証し、エラーがあれば開発者コンソールへ出力すること。
- `min`: 数値型データの最小値制限（例: `step` は 0 以上の整数、等）。
- `nonEmpty`: 必須文字列フィールドの非空チェック。
- `regex`: 正規表現パターンへの適合確認（例: カラーコードが `#` から始まる16進表現、等）。
- `in`: 許容する値（enumリスト）にマッチするか（例: `type` は `[1, 2, 3]` のいずれか、等）。

## 3. レースルールおよびデータ定義仕様

### 3.1 コース仕様
* **コース全長**: **70マス**。
* **ゴール判定**: いずれかの馬の座標（位置）が **70マスを超えた（71以上になった）** 時点でゴールインと判定。
* **順位確定**: 1着および2着が確定した時点でレースが即時終了。

### 3.2 モンスター（出走馬）仕様
レースは5頭立てで実施され、以下の固有カラーとカラーコードが割り当てられます。

| ID | モンスター名 | テーマカラー | カラーコード |
| :---: | :--- | :---: | :---: |
| **1** | サラマンダー (Salamander) | レッド | `#FF0000` |
| **2** | フェニックス (Phoenix) | オレンジ | `#FFA500` |
| **3** | グリフィン (Griffin) | グリーン | `#008000` |
| **4** | ケルピー (Kelpie) | ブルー | `#0000FF` |
| **5** | ワイバーン (Wyvern) | パープル | `#800080` |

### 3.3 ベット・オッズ仕様
ゲームは1着・2着の順不同組み合わせを予想する「**馬連 (Quinella)**」を採用します。以下の10通りのオッズが規定されています。

| 組み合わせ | オッズ倍率 | 組み合わせ | オッズ倍率 |
| :---: | :---: | :---: | :---: |
| **1 - 2** | **5倍** | **2 - 4** | **11倍** |
| **1 - 3** | **7倍** | **2 - 5** | **15倍** |
| **1 - 4** | **10倍** | **3 - 4** | **13倍** |
| **1 - 5** | **14倍** | **3 - 5** | **17倍** |
| **2 - 3** | **8倍** | **4 - 5** | **20倍** |

## 4. プレイカード（山札・効果）仕様

山札は全 **60枚** のカードで構成され、シャッフルされた状態でゲームがスタートします。

### 4.1 カード詳細効果

#### ① ステップカード (Step Card) - 計45枚
特定の馬をダイレクトに指定マス分進める。

* **ドラゴン (Red)**: `+5`, `+9`, `+10` (各3枚)
* **デーモン (Orange)**: `+5`, `+6`, `+8` (各3枚)
* **ドラキー (Green)**: `+4`, `+5`, `+7` (各3枚)
* **ゴーレム (Blue)**: `+4`, `+5`, `+6` (各3枚)
* **ゴースト (Purple)**: `+3`, `+4`, `+5` (各3枚)

#### ② 順位カード (Rank Card) - 計13枚
現在の順位に位置する馬を進める。

* **1位**: `+5`, `+10`, `+15` (各1枚)
* **2位**: `+5`, `+10`, `+15` (各1枚)
* **3位**: `+5`, `+10`, `+15` (各1枚)
* **4位**: `+5`, `+10`, `+15` (各1枚)
* **最下位 (Last)**: `+35` (1枚)

> [!IMPORTANT]
> **【重要】同着無効ルール**
> 指定された順位の座標に、同着（同じマス）の馬が2頭以上並んでいる場合、対象の順位を特定できないため、**カードの効果は無効（前進は0マス）** となる。

#### ③ ダッシュカード (Dash Card) - 計2枚 (各1枚)
レース状況を計算して移動量を決める特殊カード。

##### 1. ブースト (Boost / ID: 1)
* **対象**: 単独1位の馬。
* **移動量**: $(\text{1位の座標} - \text{2位の座標}) \times 2$
* **条件**: 1位および2位が同着でない（単独である）こと。

##### 2. キャッチアップ (Catch Up / ID: 2)
* **対象**: 単独2位の馬。
* **移動量**: $(\text{1位の座標} - 1) - \text{2位の座標}$
* **条件**: 1位および2位が同着でない（単独である）こと。2位の馬が、1位の馬のちょうど1マス後ろにワープするように移動する。

## 5. 非機能要件 (Non-Functional Requirements)

### 5.1 ユーザビリティおよび描画整合性
* **FPSの安定性**: 画面描画は `requestAnimationFrame` に追従し、通常60FPSを維持すること。重い計算を避けるため、DOM挿入は `DocumentFragment` を介して一括でバッチ描画を実行する。
* **イージング補間**: 可変描画レートによる馬の移動のガタつきを防ぐため、物理更新の周期と描画の余剰時間（デルタ値）を用いて、位置変化を線形補間（スムーズスクロール）すること。

### 5.2 セキュリティ・安全性
* **エスケープ処理**: HTMLテンプレートエンジン (`template.js`) で変数を展開する際、入力された文字列内の HTMLタグ・スクリプトを自動エスケープし、XSS（クロスサイトスクリプティング）を防止すること。

### 5.3 パフォーマンス
* **ファイルサイズ制限**: 圧縮前の主 JavaScript コード（`main-min.js`）および関連ファイルの合計サイズが150KB以下になるように極力軽量に保つこと。
